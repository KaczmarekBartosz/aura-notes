## 1. @code_rams — Agentic Engineering Takeaways
**Date:** 2026-02-13 | [Link](https://x.com/code_rams/status/2022459737648865415)

Key insights from Lex Fridman × Peter Steinberger conversation:
- Agentic engineering > vibe coding (clean architecture beats prompt tricks)
- Start stupid simple (OpenClaw began as 1-hour WhatsApp→CLI relay)
- More prompts ≠ more intelligence (small systems, tight loops)
- Self-modifying agents already here (inspect tools/docs/source)
- Fix forward mindset (patch quickly, re-run, keep main deployable)
- Voice prompting underused (conversational > structured dumps)
- Treat agents like capable engineers with short memory
- Run agents in parallel (features/bugs/docs/tests)
- Security is not optional (sandboxing, allowlists, scoped permissions)
- Models have personalities (pick by workflow fit, not hype)
- Shift from app-centric → agent-centric UX
- Builder role shift: coding → architecture/constraints/outcomes
